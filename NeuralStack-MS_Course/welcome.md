## Welcome

This course is built for engineers, analysts, and career-changers who understand how software systems work but are new to securing AI systems specifically. No prior security certification is required. Some familiarity with how machine learning models are built and deployed will help, but every technical term is defined the first time it appears.

By the end of this course, you will be able to:

- Explain why AI systems require a different threat model than traditional software
- Describe the ten risk categories in the OWASP Top 10 for LLM Applications and recognize them in real systems
- Identify where data poisoning, supply chain compromise, and provenance failures can occur in an ML pipeline
- Map the attack surface of a RAG (Retrieval-Augmented Generation) pipeline, from ingestion to generation
- Distinguish prompt injection from jailbreaking and describe baseline mitigations for each
- Speak the vocabulary of AI governance frameworks (NIST AI RMF, ISO/IEC 42001, MITRE ATLAS) well enough to navigate them
- Plan your next concrete steps into AI security engineering as a career path

**Structure:** Six modules, each with a lesson, key takeaways, a hands-on exercise, and a short quiz with an answer key. A comprehensive review closes the course. A glossary and resource list are included at the end for reference.

**How to use it:** Work through the modules in order — each builds on the last. Don't skip the exercises; they're where the concepts actually stick.
